Introduction to Firewalls

A firewall is a network security system that monitors and controls traffic between a trusted network and an untrusted network (e.g., the internet). It prevents unauthorized access, blocks malicious traffic, and enforces security policies.

Real‚Äëworld examples
  • Company perimeter firewall protecting office networks.
  • Home router firewall preventing inbound unsolicited connections.
  • University firewalls segmenting research networks from student VLANs.

Firewall Functions

Traffic Filtering

Examines packets based on IPs, ports, and protocols. Can be stateless or stateful.

  • Block inbound TCP 22; allow outbound 80/443.
  • Stateful tracking validates connection sequences.
Access Control

Controls who/what can reach resources via authentication, IP allow‚Äëlists, least‚Äëprivilege.

NAT & Proxy Services

NAT hides internal addresses and enables shared public IPs. Proxy intermediates requests for caching, filtering, logging.

Learn more: Network address translation

Intrusion Prevention (IPS)

Deep packet inspection detects blocks: scans, exploits, DoS. Signatures and anomaly detection.

MCQ: Which function hides internal network structure?

Answer: Network Address Translation (NAT).

Types of Firewalls

Track connection state across layers; common in modern OS/devices. Examples: pfSense, OPNsense.

Mitigation Strategies (Risk Treatment)

Avoidance

Eliminate the risky activity altogether (e.g., decomission outdated FTP services, do not launch insecure apps).

Reduction (Mitigation/Control)

Lower likelihood/impact with controls: patching, MFA, firewalls/IDS/IPS, encryption, segmentation, backups.

Sharing (Transfer)

Shift financial/operational consequences to third parties (cyber insurance, MSSP/SOC, cloud SLAs).

Acceptance

Formally accept residual risk when cost > impact or probability is very low; document and review regularly.
